# Contribution Workflow

## Branching guidance

- use focused feature branches
- open PRs with a clear vertical slice
- avoid mixing architecture refactors with feature shipping unless necessary

## Documentation expectation

Every meaningful new domain surface should add or update:
- a domain doc
- a diagram if behavior changed materially
- an ADR if a major architectural decision changed

## Commit hygiene

Prefer commits that explain intent, not just file movement.
