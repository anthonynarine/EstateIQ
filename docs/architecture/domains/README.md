# Domains

Each domain follows a reusable pattern:

- `README.md` — entry point and ownership summary
- `01-domain-overview.md` — what the domain is for
- `02-data-model.md` or equivalent — the core shape
- `03-request-lifecycle.md` or equivalent — request and mutation flow
- `04-reporting-impact.md` — how the domain contributes to reporting
- `05-future-evolution.md` — safe growth path
